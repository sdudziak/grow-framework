# Contributing to GROW

GROW is public because a framework like this gets better the more teams run it and
push back on it. If you use it and something rubs wrong, that feedback is the contribution.

## Ways to help

- **Open an issue** — a category that doesn't fit your context, a level threshold that feels
  off, a scoring edge case, an unclear instruction, a translation gap.
- **Open a PR** — a sharper criterion, a fix, a new language, a worked example, a clarification
  in the README.
- **Just tell your story** — how it landed on your team, what you had to adapt. That shapes the
  next version even without a code change.

No contribution is too small. Fixing one confusing sentence counts.

## License

The framework is licensed **CC BY 4.0**. By contributing, you agree your contribution is
released under the same license. That's the whole deal — no CLA, no paperwork.

## Credit (opt-in)

If a contribution meaningfully improves the framework, it can be credited in the authors list.
That is **opt-in, not automatic** — tick the box in your issue or PR when you'd like it:

```
- [ ] I'd like to be credited in the authors list for this contribution
```

No box, no name. Contribute for the karma; the credit is there only if you want it.
